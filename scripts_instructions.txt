# 📜 WSMS COMPONENT DOCUMENTATION

This file provides a technical breakdown of the individual modules located in the /scripts directory.

---
1. server-health-audit.sh
-------------------------
Purpose: Core infrastructure monitoring.
Metrics: CPU Load, RAM utilization, Disk I/O, and service uptime (Nginx/PHP/SQL).
Output: Color-coded health dashboard with automated operational advice.

2. wp-fleet-status-monitor.sh
-----------------------------
Purpose: Application-level observability.
Metrics: Core versions, plugin counts, and update status across the entire fleet.
Context: Multi-tenant aware (operates within isolated system-user contexts).

3. wp-automated-maintenance-engine.sh
-------------------------------------
Purpose: Fleet-wide patching orchestration.
Logic: Updates Core, Plugins, and Themes. Includes database migrations and post-patch cache invalidation.

4. infrastructure-permission-orchestrator.sh
--------------------------------------------
Purpose: Security & Isolation.
Logic: Hardens filesystem permissions (755/644) and implements ACLs for the 'ubuntu' backup operator.

5. wp-full-recovery-backup.sh
-----------------------------
Purpose: Disaster Recovery (Snapshot).
Logic: Creates a full bare-metal archive of the entire web root and database. 35-day retention.

6. wp-essential-assets-backup.sh
--------------------------------
Purpose: High-frequency Asset Backup.
Logic: Targets only unique data (Uploads, Themes, Plugins). Optimized for storage efficiency. 14-day retention.

7. mysql-backup-manager.sh
--------------------------
Purpose: Database Reliability.
Logic: Dynamically extracts credentials from wp-config.php and performs production-safe mysqldumps.

8. nas-sftp-sync.sh
-------------------
Purpose: Hybrid Cloud Data Integrity.
Logic: Uplinks local archives to a remote Synology NAS via secure SFTP. Implements off-site retention policies.

9. red-robin-system-backup.sh
-----------------------------
Purpose: Emergency System Recovery.
Logic: Backs up OS-level configurations and critical metadata for rapid server reconstruction.

10. wp-interactive-backup-tool.sh
---------------------------------
Purpose: On-demand Operations.
Interface: A menu-driven CLI utility for manual, high-stakes backup tasks.

11. wsms-alias-setup.sh & wsms-cron-scheduler.sh
------------------------------------------------
Purpose: Environment Provisioning.
Logic: Automates shell configuration (aliases) and crontab scheduling for unattended operations.

12. wp-help.sh
--------------
Purpose: Internal Documentation.
Interface: Centralized command reference for all WSMS aliases and functions.
