# WSMS PRO v4.3 - Deployment Guide

## Prerequisites

- Ubuntu 20.04+ or 22.04+
- sudo access
- WordPress sites with valid `wp-config.php`

## Install (English)

```bash
git clone https://github.com/maleclukas-prog/wp-server-management-system.git
cd wp-server-management-system
nano installers/install_wsms.sh
chmod +x installers/install_wsms.sh
./installers/install_wsms.sh
```

## Install (Polish)

```bash
nano installers/install_wsms_pl.sh
chmod +x installers/install_wsms_pl.sh
./installers/install_wsms_pl.sh
```

## Post-Install Checks

```bash
source ~/.bashrc
wp-status
wp-help
wp-hosts-sync
```

`wp-hosts-sync` updates a managed block in `/etc/hosts` for all domains from `SITES` and may request sudo.

For Fish:

```fish
source ~/.config/fish/config.fish
wp-status
```

## Key Paths

- `~/scripts/` runtime modules generated by installer
- `~/scripts/wsms-config.sh` generated central configuration
- `~/logs/wsms/` categorized logs
- `/etc/hosts` managed WSMS local-domain block (created by `wp-hosts-sync`)

## Runtime Preview Export (No Install)

To inspect installer-generated runtime modules as separate, copy-ready files without running installation:

```bash
bash tools/wsms-export-runtime-scripts.sh
```

Generated preview directories:

- `scripts/runtime-preview/en/`
- `scripts/runtime-preview/pl/`

Note: preview files are generated from installers and are git-ignored. The installer remains the source of truth.

## Uninstall

```bash
./tools/wsms-uninstall.sh --dry-run
./tools/wsms-uninstall.sh
./tools/wsms-uninstall.sh --force
```

Notes:

- `--dry-run` previews actions without changing files.
- Uninstall also removes the WSMS marker block from `/etc/hosts`.
