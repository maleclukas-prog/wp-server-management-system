# Repository scripts directory

Runtime WSMS modules are generated by installers into `~/scripts/` on target hosts.

This repository directory is intentionally metadata-only to avoid drift between committed snapshots and installer-generated runtime modules.

If you want inspectable/copy-ready module files without running installer, generate previews from installers:

```bash
bash tools/wsms-export-runtime-scripts.sh
```

To export only selected modules:

```bash
bash tools/wsms-export-runtime-scripts.sh --only wp-automated-maintenance-engine.sh,wp-smart-retention-manager.sh,wp-help.sh
```

Generated preview paths:

- `scripts/runtime-preview/en/`
- `scripts/runtime-preview/pl/`
